import ExpressionPlayground from "../../components/ExpressionPlayground.tsx";

This playground runs stringent's **runtime engine** (`safeParse` +
`evaluateAst`) right in your browser, against the quickstart grammar from
[Getting Started](/stringent/guides/getting-started/): number and string
literals, variables and dotted paths, parentheses, `==`, `+` (overloaded for
numbers and strings), `*`, and a lazy ternary.

Type an expression — parse errors show the exact position and what would have
been valid there.

<ExpressionPlayground client:load />

:::tip[The other engine]
What you see here is the *runtime* half of stringent. The same grammar also
drives the **type-level engine**, which checks string literals in your code at
compile time — that part you can experience in the hover-able code blocks in
the guides, or in your own editor.
:::
