```ts twoslash
import {
  defineNode, number, string, path, lhs, rhs, expr, constVal,
  sameAs, fromBinding, createParser,
} from "stringent";

const numberLit = defineNode({ name: "num", pattern: [number()], precedence: "atom" });
const stringLit = defineNode({ name: "str", pattern: [string(['"', "'"])], precedence: "atom" });
const variable = defineNode({ name: "var", pattern: [path()], precedence: "atom" });

const eq = defineNode({
  name: "eq",
  pattern: [lhs().as("left"), constVal("=="), rhs(sameAs("left")).as("right")],
  precedence: 1,
  resultType: "boolean",
  eval: ({ left, right }) => left === right,
});

const parser = createParser([numberLit, stringLit, variable, eq] as const);
// ---cut---
const schema = { values: { password: "string", confirmPassword: "string" } } as const;
const values = { values: { password: "hunter2", confirmPassword: "hunter2" } };

const result = parser.evaluate("values.password == values.confirmPassword", schema, values);
//    ^?
```

**Hover the code above.** Every type on this page is real — produced by the
actual library, at documentation build time, from a grammar *you* define.

## One grammar. Two engines.

You declare each rule once — as data. The same declaration drives a
**type-level parser** that checks string literals in your code, and a
**runtime parser** that checks strings from users and config files.

<Tabs>
  <TabItem label="Compile time — string literals" icon="approve-check">
    Result types are inferred *through* the grammar — the same overloaded `+`
    yields `number` or `string` depending on its operands, and expressions
    that don't parse **don't compile**:

    ```ts twoslash
    // @errors: 2345
    import {
      defineNode, number, string, path, lhs, rhs, expr, constVal,
      sameAs, fromBinding, createParser,
    } from "stringent";

    const numberLit = defineNode({ name: "num", pattern: [number()], precedence: "atom" });
    const stringLit = defineNode({ name: "str", pattern: [string(['"', "'"])], precedence: "atom" });
    const variable  = defineNode({ name: "var", pattern: [path()], precedence: "atom" });

    const eq = defineNode({
      name: "eq",
      pattern: [lhs().as("left"), constVal("=="), rhs(sameAs("left")).as("right")],
      precedence: 1,
      resultType: "boolean",
      eval: ({ left, right }) => left === right,
    });

    const add = defineNode({
      name: "add",
      pattern: [lhs(["number", "string"]).as("left"), constVal("+"), rhs(sameAs("left")).as("right")],
      precedence: 2,
      associativity: "left",
      resultType: fromBinding("left"),
      eval: ({ left, right }) => (left as any) + (right as any),
    });

    const mul = defineNode({
      name: "mul",
      pattern: [lhs("number").as("left"), constVal("*"), rhs("number").as("right")],
      precedence: 3,
      associativity: "left",
      resultType: "number",
      eval: ({ left, right }) => left * right,
    });

    const ternary = defineNode({
      name: "ternary",
      pattern: [
        lhs("boolean").as("cond"), constVal("?"),
        expr().as("then"), constVal(":"),
        rhs(sameAs("then")).as("else"),
      ],
      precedence: 0,
      resultType: fromBinding("then"),
      lazy: true,
      eval: ({ cond, then, else: alt }) => (cond() ? then() : alt()),
    });

    const parser = createParser(
      [numberLit, stringLit, variable, ternary, eq, add, mul] as const
    );
    // ---cut---
    const seven = parser.evaluate("1+2*3", {}, {});
    //    ^?

    const greeting = parser.evaluate("'Hello '+name", { name: "string" }, { name: "Ada" });
    //    ^?

    const label = parser.evaluate("1==2 ? 'yes' : 'no'", {}, {});
    //    ^?

    // ...and an expression that doesn't parse doesn't compile:
    parser.parse("1+'a'", {});
    ```
  </TabItem>
  <TabItem label="Runtime — dynamic strings" icon="warning">
    The same grammar parses untrusted input with `safeParse()`, which never
    throws — you get an error code, an exact position, and what would have
    been valid there. These are the engine's real error objects:

    ```ts
    parser.safeParse("price * quantitty", { price: "number", quantity: "number" });
    // {
    //   code: "TYPE_MISMATCH",
    //   message: "Expected a number expression at position 8, got unknown ('quantitty' is not in the schema)",
    //   position: 8,
    //   expected: ["number"],
    //   found: "\"quantitty\"",
    // }

    parser.safeParse("price *", schema);
    // { code: "PARSE_ERROR", position: 7,
    //   message: "Expected identifier or number at position 7, found end of input" }

    parser.safeParse("price * 2 oops", schema);
    // { code: "UNEXPECTED_INPUT", position: 10,
    //   message: "Unexpected input at position 10: found \"oops\" (expected \"*\")" }
    ```

    Parse once, evaluate as often as you like:

    ```ts
    const result = parser.safeParse(userInput, schema);
    if (result.success) {
      const value = parser.evaluateAst(result.ast, values);
    }
    ```
  </TabItem>
</Tabs>

<CardGrid stagger>
  <Card title="Compile-time validation" icon="approve-check">
    `parse()` only accepts string literals that fully parse against your
    grammar. An invalid expression — a syntax error, a type mismatch, even a
    typo in a schema leaf — is a **compile-time error**, and valid expressions
    get an exactly inferred AST and result type.
  </Card>
  <Card title="Runtime safety" icon="warning">
    Dynamic strings go through `safeParse()`, which never throws for input.
    You get structured errors with an error code, a 0-based position, and the
    tokens that would have been valid there.
  </Card>
  <Card title="Polymorphic grammars" icon="puzzle">
    Union constraints, `sameAs()` and `fromBinding()` give you overloaded
    operators without per-type node variants — one `add` node handles
    `number + number → number` and `string + string → string`.
  </Card>
  <Card title="Secure by default" icon="seti:lock">
    Expressions are treated as untrusted input. All identifier and path
    lookups are own-property only — `__proto__`, `constructor` and friends
    never resolve to prototype internals, at parse time or eval time.
  </Card>
</CardGrid>
